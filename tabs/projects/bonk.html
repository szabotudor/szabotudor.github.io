bonk

bonk is a build tool for Rust projects with support for cross-compiling and publishing to all major platforms and stores. It was developed during my internship at Traverse Research.

The goal is to hide all the platform-specific tooling — Android SDK, iOS codesigning, MSIX packaging, AppImage creation — behind a unified CLI, so building and deploying to any target is a single command regardless of the host machine.

Device Abstraction

The core of bonk's cross-platform support is the device abstraction. Instead of spreading platform checks throughout the codebase, every device — Android phone, iPhone, Linux host — implements the same interface through a backend enum. 

#[derive(Clone, Debug)]
enum Backend {
    Adb(Adb),
    IDevice(IDevice),
    Imd(IMobileDevice),
    Host(Host),
}

pub struct Device {
    backend: Backend,
    id: String,
}

impl Device {
    pub fn list() -> Result<Vec<Self>> {
        let mut devices = vec![Self::host()];
        if let Ok(adb) = Adb::new() {
            adb.devices(&mut devices)?;
        }
        if let Ok(imd) = IMobileDevice::which() {
            imd.devices(&mut devices).ok();
        }
        if let Ok(idevice) = IDevice::new() {
            idevice.devices(&mut devices)?;
        }
        Ok(devices)
    }

    pub fn platform(&self) -> Result<Platform> {
        match &self.backend {
            Backend::Adb(adb)         => adb.platform(&self.id),
            Backend::Host(host)       => host.platform(),
            Backend::Imd(imd)         => imd.platform(&self.id),
            Backend::IDevice(idevice) => idevice.platform(&self.id),
        }
    }
}

The enum-based dispatch keeps all platform branching in one place. Adding a new device backend means adding one variant and implementing the small set of methods — everything downstream works without changes.

Modular Packaging

Each target platform's packaging format lives in its own crate: apk for Android, appbundle for iOS, appimage for Linux, msix for Windows, and flat-package for macOS. This keeps each packager independently usable without pulling in the full bonk CLI.

bonk coordinates them: it compiles for the right target triple, fetches any missing SDKs through the download manager, then hands off to whichever packaging crate matches the device. 

// Each platform packager is a separate crate in the workspace
// apk/      appbundle/      appimage/
// msix/     flat-package/   mvn/

// The build command resolves the packager from the device's platform
match device.platform()? {
    Platform::Android => apk::build(&env, &artifact, &config)?,
    Platform::Ios     => appbundle::build(&env, &artifact, &config)?,
    Platform::Linux   => appimage::build(&env, &artifact, &config)?,
    Platform::Windows => msix::build(&env, &artifact, &config)?,
    Platform::MacOs   => flat_package::build(&env, &artifact, &config)?,
    Platform::Host    => { /* no packaging needed */ }
}

Multi-Store Publishing

Beyond building, bonk can publish directly to Google Play, Apple App Store, Steam, Microsoft Store, Epic Games Store, and more. Credentials and signing keys are managed through bonk's settings, so the full build-sign-publish pipeline is a single command.

Diagnostics

Getting cross-compilation toolchains set up is notoriously painful. bonk doctor checks every required external tool and reports versions, making it easy to see exactly what's missing before a build fails halfway through.

Magma Engine

Magma Engine is a custom game engine written in C/C++.

It uses ImGui to display the editor interface, making it easy to extend with custom plugins and add-ons.

Powering the engine is the SystemManager, designed to allow adding to, or modifying the engine's systems.
The SystemManager makes creating new systems a simple task.

Make a class inheriting from System as a base.
Then add it to the engine by creating the system.

This design ensures extendability and compartmentalization of systems.

The API is minimal and simple to use, almost no boilerplate needed.
Or even better, replace existing systems at any time outside of that system's update functions.

Don't like the renderer?
Or the ECS?
Or the physics system?
Or you just miss a specific feature that you'd like.
Magma also has an API for creating editor windows.

Just like systems, different types of windows can be added to the editor (when available).

The base Magma code uses both the SystemManager and the EditorWindow for its built-in systems, so everything is modular.
The engine is multi-platform.
It works on Windows and Linux, and it can build games for both platforms.

Porting the engine to other platforms is easy thanks to its modularity.

Everything a user might interact with is abstracted: the filesystem, the graphics and compute API etc.
Everything is modifiable and replaceable.

The graphics/compute backend is a simple wrapper for any graphics API (in this case, OpenGL).
Changing it is a drop-in replacement, just place a different backend library inside the "shared" folder, and the engine will pick it up.

The backend is a simple interface, following this standard.
